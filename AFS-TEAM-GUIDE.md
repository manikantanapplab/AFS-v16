# AFS — Team Guide

Step-by-step reference for developers joining a project built on the Agency Frontend System (AFS).
Read this once before you start. Refer back when you need it.

---

## 1. Prerequisites

Make sure these are installed on your machine before anything else:

```bash
node --version   # v18 or higher required
npm --version    # v9 or higher required
git --version
```

Install global CLI tools if missing:

```bash
npm install -g pug-cli
npm install -g sass
```

---

## 2. Getting the Project

### Joining an existing project

```bash
git clone https://github.com/YOUR_ORG/PROJECT_NAME.git
cd PROJECT_NAME
npm install
npm run dev
# → http://localhost:3000
```

### Starting a brand new project from the template

```bash
npx degit your-github-username/agency-frontend-system my-project
cd my-project
npm install
npm run dev
```

Or go to the template repo on GitHub → **"Use this template"** → **"Create a new repository"**

> **Never clone the AFS template repo directly into a client project.**
> Use degit or "Use this template" — these give you a clean copy with no
> connection to the template's git history.

---

## 3. The Golden Rules

Before anything else, understand these:

| Rule | Why |
|------|-----|
| Work only inside `src/` | Everything else is auto-generated or system files |
| Never edit `dist/` manually | It gets overwritten on every build |
| Never edit `_base.pug` directly | Regenerated by `build-pug.mjs` on every run |
| Never edit `app.scss` directly | Regenerated by the build system |
| Never edit files in `system/` | Build scripts — changes will be overwritten |
| One dev per component | Prevents merge conflicts |
| One dev per page | Prevents merge conflicts |
| `src/sass/tokens/` — lead dev only | Token changes affect the entire project |

## 4. Folder Structure

```
afs.config.mjs          ← project config — set MODE here

src/
  assets/
    images/             ← drop images here
    fonts/

  components/           ← one folder per UI component
    hero/
      _hero.pug         ← Pug mixin (the template)
      hero.scss         ← styles
      hero.preview.html ← standalone preview in browser

  layouts/
    _base.pug           ← AUTO-GENERATED — never edit
    _header.pug         ← edit: logo, nav links
    _footer.pug         ← edit: copyright, footer links
    header.scss         ← header styles

  pages/
    index.pug           ← one file per page
    about.pug

  sass/
    tokens/
      _colors.scss      ← brand colors — edit at project start
      _typography.scss  ← font family — edit at project start
      _spacing.scss     ← spacing scale

    base/               ← global styles — do not edit
    utilities/          ← mixins, functions — do not edit
    pages/
      about.scss        ← page-specific styles go here

  js/
    components.js       ← Alpine.js registrations
    swiper-init.js      ← Swiper auto-initializer

dist/                   ← compiled output — do not edit, do not commit
system/                 ← build scripts — do not edit
```

## 5. Daily Workflow

### Starting your day

```bash
git pull           # always pull latest before you start
npm run dev        # starts dev server at http://localhost:3000
```

### Creating a new component

```bash
npm run new my-component
```

This scaffolds `src/components/my-component/` with three files:
- `_my-component.pug` — the Pug mixin
- `my-component.scss` — styles (pre-wired with `@use` and `@layer`)
- `my-component.preview.html` — open directly in a browser to preview

### Creating a new page

```bash
npm run page my-page
```

This creates:
- `src/pages/my-page.pug` — the page template
- `src/sass/pages/my-page.scss` — page-specific styles

### Adding images

Drop files into `src/assets/images/` — they auto-copy to `dist/assets/images/`
while dev is running. Reference them in Pug:

```pug
img(src="assets/images/hero.jpg" alt="Hero image")
img(src="assets/images/logo.svg" alt="Logo")
```

### Ending your day

```bash
git add src/
git commit -m "feat: describe what you built"
git push
```

## 6. Writing Pug (HTML)

### Page template anatomy

```pug
extends ../layouts/_base.pug

//- These two blocks are AUTO-MANAGED by build-pug.mjs
//- Do not add or remove links here manually
block pagecss
  link(rel="stylesheet" href="assets/css/pages/about.css")
block componentcss
  link(rel="stylesheet" href="assets/css/components/hero.css")

block title
  | About | Company Name

block content
  //- Include component mixin files at the top of block content
  include ../components/hero/_hero.pug
  include ../components/card/_card.pug

  //- Then call the mixins
  +hero({
    title: 'About us',
    text:  'A short description.'
  })
```

### Using a component — two steps

**Step 1** — include the mixin file:
```pug
block content
  include ../components/card/_card.pug
```

**Step 2** — call the mixin:
```pug
  +card({
    title: 'Card Title',
    text:  'Description here.',
    image: 'assets/images/card.jpg',
    link:  '/services.html'
  })
```

### Pug syntax quick reference

```pug
//- Plain text output
h1= data.title
p= data.text

//- Interpolation inside text
p Welcome, #{user.name}!

//- Attribute
a(href=data.link class="btn btn-primary")= data.label

//- Dynamic class
section(class=data.dark ? 'section-dark' : '')

//- Conditional block
if data.image
  img(src=data.image alt=data.alt)

//- Else
if data.title
  h2= data.title
else
  h2 Default Heading

//- Loop
each item in items
  li.nav-item
    a(href=item.link)= item.label

//- Loop with index
each item, i in items
  .card(class=i === 0 ? 'is-first' : '')

//- Local variable
- var count = items.length
p #{count} items found

//- Bootstrap grid
.container
  .row.g-4
    .col-md-6.col-lg-4
      p Column content

//- Comment (not compiled to HTML)
//- This is a Pug comment

//- HTML comment (compiled to HTML)
<!-- This appears in the HTML output -->
```

### Mixin blocks (slot content)

Some mixins accept a `block` for custom content:

```pug
+hero({ title: 'Hello' })
  //- Content here goes into the block
  .hero-badge New feature

+card({ title: 'Card' })
  button.btn.btn-sm.mt-3 Custom button
```

## 7. Writing SCSS

### The required file header

Every component SCSS file must start with exactly this:

```scss
@use '../../sass/utilities/mixins' as *;

@layer components {
  .my-component {
    // your styles here
  }
}
```

- `@use` loads `rem()`, `fluid-steps()`, and flex helpers
- `@layer components` ensures your styles override Bootstrap — never use `!important`

### Using design tokens

Always use tokens for color, spacing, and typography:

```scss
.my-component {
  // ✅ correct — use tokens
  color:            var(--color-text-primary);
  background:       var(--color-bg-subtle);
  border:           1px solid var(--color-border);
  font-size:        var(--text-lg);
  padding-block:    var(--section-padding-y);

  // ✅ use rem() for values from Figma not in the scale
  border-radius:    rem(12);
  gap:              rem(20);
  max-width:        rem(640);

  // ❌ avoid — hard-coded values break theming and dark mode
  color:            #212121;
  font-size:        18px;
  padding:          48px;
}
```

### Token reference

**Colors:**
```scss
var(--color-primary)           // main brand color
var(--color-secondary)         // accent color
var(--color-text-primary)      // main body text
var(--color-text-secondary)    // secondary / muted text
var(--color-bg-body)           // page background
var(--color-bg-subtle)         // light-tinted background
var(--color-border)            // standard border
var(--color-border-soft)       // 50% opacity border
```

**Typography:**
```scss
var(--text-xs)      // 12px     var(--font-light)    // 300
var(--text-sm)      // 14px     var(--font-regular)  // 400
var(--text-base)    // 16px     var(--font-semi)     // 600
var(--text-lg)      // 18px     var(--font-bold)     // 700
var(--text-xl)      // 20px
var(--text-2xl)     // 24px
var(--text-3xl)     // 30px
var(--text-4xl)     // 36px
```

**Spacing:**
```scss
var(--space-4)           //  4px
var(--space-8)           //  8px
var(--space-16)          // 16px
var(--space-24)          // 24px
var(--space-32)          // 32px
var(--space-48)          // 48px
var(--space-64)          // 64px
var(--section-padding-y) // fluid: clamp(2rem, 5vw, 5rem)
```

### Utility classes — use in Pug without writing SCSS

```pug
//- Font sizes (number = px value, fs-10 to fs-100)
h2.fs-32       //→ font-size: 2rem
p.fs-14        //→ font-size: 0.875rem
span.fs-12     //→ font-size: 0.75rem

//- Gaps (number = px value, g-1 to g-50)
.d-flex.g-8    //→ gap: 0.5rem
.d-flex.g-16   //→ gap: 1rem
.d-flex.g-24   //→ gap: 1.5rem

//- Square size (width + height)
.icon.size-24  //→ width: 1.5rem; height: 1.5rem
.btn.size-48   //→ width: 3rem; height: 3rem
```

### `fluid-steps()` — responsive scaling

```scss
// Scale font-size from 32px on mobile to 72px on desktop
@include fluid-steps(font-size, (32px, 72px));

// Scale padding at three breakpoints
@include fluid-steps(padding-block, (24px, 48px, 80px));
```

## 8. Alpine.js Interactivity

All components are globally registered — use `x-data` on any element, no imports needed.

### Quick inline state (no registration needed)

```pug
//- Toggle
div(x-data="{ open: false }")
  button(@click="open = !open") Toggle
  div(x-show="open") Hidden content

//- Counter
div(x-data="{ count: 0 }")
  button(@click="count++") +
  span(x-text="count")
```

### Registered components

```pug
//- Navbar scroll state — adds .is-scrolled to nav on scroll
nav.site-header(x-data="navbar" :class="{ 'is-scrolled': scrolled }")

//- Dark/light mode toggle — persists in localStorage
button(x-data="themeToggle" @click="toggle")
  i.bi(x-show="!dark" class="bi-moon")
  i.bi(x-show="dark" class="bi-sun")

//- Tabs — pass default active index
div(x-data="tabs(0)")
  button(@click="setTab(0)" :class="{ active: isActive(0) }") Tab 1
  button(@click="setTab(1)" :class="{ active: isActive(1) }") Tab 2
  div(x-show="isActive(0)") Content 1
  div(x-show="isActive(1)") Content 2

//- Accordion
div(x-data="accordion")
  button(@click="toggle(0)") Question 1
  div(x-show="isOpen(0)") Answer 1
  button(@click="toggle(1)") Question 2
  div(x-show="isOpen(1)") Answer 2

//- Animated number counter (triggers on scroll into view)
//- counter(targetValue, durationMs)
div(x-data="counter(250, 1500)")
  span(x-text="value")

//- Modal
div(x-data="modal")
  button(@click="open()") Open modal
  div(x-show="isOpen" @click.self="close()")
    .modal-box
      button(@click="close()") ✕
      p Modal content here

//- Form submit with loading/success/error
div(x-data="form" @submit.prevent="submit")
  input(type="text" x-model="fields.name" name="name")
  input(type="email" x-model="fields.email" name="email")
  button(@click="submit" :disabled="loading")
    span(x-show="!loading") Send
    span(x-show="loading") Sending...
  p(x-show="success") Sent successfully.
  p(x-show="error") Something went wrong.
```

## 9. Swiper Component

```pug
include ../components/swiper/_swiper.pug

-
  var slides = [
    {
      image: 'assets/images/slide-1.jpg',
      alt:   'Slide one',
      tag:   'Category',
      title: 'Slide Title',
      text:  'Supporting text for this slide.',
      btn:   { label: 'Learn more', link: '#' }
    },
    {
      image: 'assets/images/slide-2.jpg',
      alt:   'Slide two',
      title: 'Another Slide'
    }
  ]

//- Basic
+swiper({ slides: slides })

//- Responsive multi-slide
+swiper({ slides: slides, perView: 1, perViewMd: 2, perViewLg: 3, gap: 24 })

//- Fade effect with autoplay
+swiper({ slides: slides, effect: 'fade', autoplay: true, interval: 5000 })

//- Loop disabled, arrows + dots
+swiper({ slides: slides, loop: false, arrows: true, dots: true })

//- Main + thumbnail pair
+swiper({ slides: slides, id: 'main', thumbs: true, thumbsId: 'thumbs' })
+swiperThumbs({ slides: slides, id: 'thumbs' })
```

**All swiper options:**

| Option       | Type    | Default  | Description                               |
|--------------|---------|----------|-------------------------------------------|
| `slides`     | Array   | required | Array of slide objects                    |
| `id`         | String  | auto     | Unique ID — required if multiple on page  |
| `effect`     | String  | `slide`  | `slide`, `fade`, `cube`, `flip`, `cards`  |
| `perView`    | Number  | `1`      | Slides visible at once (mobile)           |
| `perViewMd`  | Number  | —        | Slides visible at 768px+                  |
| `perViewLg`  | Number  | —        | Slides visible at 1200px+                 |
| `gap`        | Number  | `0`      | Gap between slides in px                  |
| `loop`       | Boolean | `true`   | Enable infinite loop                      |
| `autoplay`   | Boolean | `false`  | Enable autoplay                           |
| `interval`   | Number  | `4000`   | Autoplay delay in ms                      |
| `arrows`     | Boolean | `true`   | Show prev/next buttons                    |
| `dots`       | Boolean | `true`   | Show pagination dots                      |
| `thumbs`     | Boolean | `false`  | Enable thumbnail navigation               |
| `thumbsId`   | String  | —        | ID of the thumbnails swiper instance      |
| `centered`   | Boolean | `false`  | Center active slide                       |

## 11. Build & Handoff to BE Team

### Dev build (with sourcemaps)

```bash
npm run build
```

### Production build (before BE handoff)

```bash
# Step 1 — set CRITICAL = true in afs.config.mjs
# Step 2 — build
npm run build:prod
# Step 3 — hand over the dist/ folder
```

**Production pipeline:**
1. Pug → HTML (CSS blocks auto-written per mode)
2. SCSS → compressed CSS with sourcemaps
3. JS + assets copied to `dist/`
4. PurgeCSS removes all unused CSS classes
5. Critical CSS inlined in `<head>` (because `CRITICAL = true`)

### What the BE team gets

Hand over the entire `dist/` folder:

```
dist/
  index.html
  about.html
  services.html
  contact.html
  assets/
    css/
      app.css              (single mode)
      base.css             (per-component mode)
      pages/               (per-page and per-component mode)
      components/          (per-component mode)
    js/
      components.js
      swiper-init.js
    images/
```

For BE integration details, see `INTEGRATION.md`.

## 12. Quick Start: New Component End-to-End

```bash
# 1. Scaffold
npm run new feature-banner
```

```pug
//- 2. src/components/feature-banner/_feature-banner.pug
mixin feature-banner(data)
  section.feature-banner(class=data.class || '')
    .container
      .feature-banner-inner
        if data.tag
          span.feature-banner-tag= data.tag
        if data.title
          h2.feature-banner-title= data.title
        if data.text
          p.feature-banner-text= data.text
        if data.btn
          a.btn.btn-primary(href=data.btn.link)= data.btn.label
        block
```

```scss
// 3. src/components/feature-banner/feature-banner.scss
@use '../../sass/utilities/mixins' as *;

@layer components {
  .feature-banner {
    padding-block: var(--section-padding-y);
    background: var(--color-bg-subtle);

    &-inner {
      display: flex;
      flex-direction: column;
      gap: rem(16);
      max-width: rem(720);
    }

    &-tag {
      font-size: var(--text-sm);
      font-weight: var(--font-semi);
      color: var(--color-primary);
      text-transform: uppercase;
      letter-spacing: 0.05em;
    }

    &-title {
      font-size: var(--text-3xl);
      font-weight: var(--font-bold);
      color: var(--color-text-primary);
    }

    &-text {
      font-size: var(--text-lg);
      color: var(--color-text-secondary);
    }
  }
}
```


## 14. Common Problems & Fixes

| Problem | Fix |
|---------|-----|
| Component CSS not loading after adding include | Restart `npm run dev` |
| New component not detected in per-component mode | Folder name, `.pug` name, and `.scss` name must all match exactly |
| Bootstrap styles overriding your component | Wrap your SCSS in `@layer components {}` |
| `rem() not found` error | Add `@use '../../sass/utilities/mixins' as *;` at top of file |
| `var(--token)` not resolving | Check for missing semicolons in `_colors.scss` or `_spacing.scss` |
| Images 404 in browser | Run `npm run copy:assets` or restart dev |
| JS not updating in browser | Run `npm run copy:js` or restart dev |
| Class disappears in prod build | Add to `safelist` in `postcss.config.js` |
| Two devs editing same component | Coordinate — one dev per component folder |
| `pug: command not found` | `npm install -g pug-cli` |
| `sass: command not found` | `npm install -g sass` |
| Port 3000 already in use | Change `server.port` in `vite.config.js` |
| Dark mode tokens not switching | Check `data-theme="dark"` is on the `<html>` tag |
